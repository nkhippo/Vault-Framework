## Summary

各プロジェクトの正式リポジトリ名、通称、機能キーワードの対応表。あいまいなアプリ名(例:「発音のアプリ」)から正式プロジェクトを特定する目的で使用する。

**このファイルは導入者が自分のプロジェクトに合わせて編集することが前提のテンプレート。**

## 使用ルール

- 新規プロジェクトを追加した際は必ずこのファイルにエントリを追加する
- 通称・機能キーワードは実際に自分が会話で使う語彙を反映する
- 曖昧さがある場合は「候補: X または Y」として複数記載してよい

## エントリテンプレート

### <your-project-name>

- 正式リポジトリ名: `<your-project-name>`
- カテゴリ: <カテゴリ、例: Webアプリ / MCP サーバ / モバイルアプリ 等>
- 通称: <通称 1>, <通称 2>, <呼び方いろいろ>
- 機能キーワード: <機能を表すキーワードを列挙>
- 対象言語: <該当する場合>
- UI 対応言語: <該当する場合>
- 関連リポジトリ: <該当する場合>
- 一言メモ: <このプロジェクトが何かを 1〜2 行で>

## メンテナンス

このファイルは Skill が「あいまい名解決」で最初に読むファイル。新規プロジェクト追加時は必ずここに追記する。追記は Cursor 経由でも導入者の手動編集でも可。

vocabulary.md の project セクションと内容が矛盾しないよう、両者を同時に更新する。

## 導入者への注意

**初回導入時に必ずカスタマイズすべきファイル**。例示コメント(HTML コメント内)は削除または残してよいが、`<your-project-name>` プレースホルダのエントリは自分の実プロジェクトで置き換える。

エントリがゼロの状態でも Skill は動作するが、あいまい名解決フローで候補が見つからない旨をユーザーに伝えることになる。
