## Summary

発信先(note / Zenn / blog 等)向け記事を Claude とあなた(導入者)が協働で書くプロセスの骨格。

## 位置づけ

- 層 1: 文体 → `writing_style.md`
- **層 2**: プロセス(このファイル)
- 層 3: 例集 → `writing_examples.md`

## 前提として読み込むもの

1. アカウント設計・発信方針ドキュメント
2. `writing_style.md` / `writing_examples.md`
3. `20_notes/published/*`(参照モデル)
4. `00_meta/profile.md`(価値判断が必要な場合)

## 全体フロー

```
題材の持ち込み
    ↓
第1段階: 概要評価(Go / Go before revise / No Go)
    ↓
第2段階: 詳細執筆
    1. 骨子案
    2. Q&A(素材の掘り下げ)
    3. 心臓部から順にドラフト
    4. 全体組み立て
    5. 推敲
    ↓
タイトル・告知文・タグ等
    ↓
あなた(導入者)が発信先へ投稿
    ↓
[トリガー] 投稿完了・URL 共有・微修正など
    ↓
Vault ミラー保存 + ガイドライン更新を Claude 側から自動実施
```

## 第 1 段階: 概要評価

題材を次の観点で評価する(導入者が観点を差し替えてよい):

1. **方針一致**: 書くテーマに入っているか
2. **実体験ベース**: 伝聞・一般論ではなく体験に根ざしているか
3. **独自性**: 既存記事群との差分を 1 行で言えるか
4. **読者リーチ**: 想定読者に何を届けるか

判定: **Go** / **Go before revise** / **No Go**。理由を明示する。

## 第 2 段階: 詳細執筆

1. **骨子案**: 記事タイプ、想定字数、章構成、使う素材。OK 後に進む
2. **Q&A**: 心臓部の章について、場面の解像度・感情の多面性・決断の背景を引き出す
3. **心臓部からドラフト**: 中盤の芯となる章から書き、前後を組む
4. **全体組み立て・推敲**: 層 1 / 層 3 に照らして調整

## Vault ミラー保存とガイドライン更新の運用

あなた(導入者)が明示的に保存依頼しなくても、次を検知したら Claude が自動実行する。

### 発動トリガー

- 発信先への投稿完了報告
- 記事 URL の共有
- タイトル・タグ確定情報
- 記事の微修正・反応の報告

### 発動時の手順

1. **記事ミラー**: `20_notes/published/YYYY-MM-DD_<slug>.md`(または導入者の公開ディレクトリ規約)
2. **chat_log 保存**: `10_chat_logs/YYYY/MM/` 等
3. **例集追記**: `writing_examples.md` に今回の指摘ペアを追記(**必須**。忘れると次回同じ指摘が繰り返される)
4. **README 維持**: ガイド索引が壊れないことを確認

### 発動しない条件

- 執筆中のドラフト段階
- 未投稿の相談・質問のみ

### メタルール

- 「保存しますか?」と問わず実行し、結果(path / id)を報告する
- 見落としを指摘されたら、その時点で手順を実行する

## 更新

新しいフローが定着したら本ファイルへ追記する。個別の語感修正は層 3 へ。
